#### **Error 2: 401 Unauthorized**
**Scenario:** Frontend requests data but Row Level Security (RLS) blocks access

**Browser Console:**
```
GET https://your-project.supabase.co/rest/v1/medications 401 (Unauthorized)
Error: new row violates row-level security policy for table "medications"
```

**Root Cause:** RLS policies not set up correctly, or user not authenticated

**Fix:** Verify RLS policies:
```sql
-- Check RLS policies on medications table
SELECT 
  schemaname,
  tablename,
  policyname,
  permissive,
  roles,
  cmd,
  qual
FROM pg_policies
WHERE tablename = 'medications';
```

**Expected:** At least 1 policy allowing `SELECT` for authenticated users

---

#### **Error 3: 403 Forbidden**
**Scenario:** User authenticated but doesn't have permission to access resource

**Browser Console:**
```
GET https://your-project.supabase.co/rest/v1/medications?user_id=eq.uuid 403 (Forbidden)
```

**Root Cause:** Multi-tenancy isolation not working (user trying to access another tenant's data)

**Fix:** Verify RLS policies filter by `tenant_id`:
```sql
-- Medications RLS policy (example)
CREATE POLICY "Users can only see their family's medications"
  ON medications
  FOR SELECT
  USING (
    tenant_id IN (
      SELECT tenant_id FROM users WHERE id = auth.uid()
    )
  );
```
