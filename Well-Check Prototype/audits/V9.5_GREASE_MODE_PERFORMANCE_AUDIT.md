## 🔧 **IDENTIFIED ISSUES & FIXES**

### **Issue 1: Z-Index Conflict**

**Problem:**
- Demo Controls: z-[9999]
- Perspective Drawer: z-50
- Demo Controls may render above Perspective Drawer backdrop, causing visual clash

**Fix:**
```tsx
// Change Demo Controls z-index from z-[9999] to z-40
<div className="fixed bottom-4 right-4 z-40">
```

**Rationale:**
- Demo Controls are debugging tools, not critical UI
- Should render below Panic Button (z-50) and Perspective Drawer (z-50)
- Still above main content (z-0) for visibility

---

### **Issue 2: Drag Constraints**

**Problem:**
- Drag constraints set to `{ top: 0, left: 0, right: 0, bottom: 0 }`
- This may allow dragging outside viewport bounds

**Fix:**
```tsx
// Use viewport dimensions for drag constraints
<motion.div
  drag
  dragConstraints={{
    top: -window.innerHeight / 2,
    left: -window.innerWidth / 2,
    right: window.innerWidth / 2,
    bottom: window.innerHeight / 2,
  }}
/>
```

**Rationale:**
- Prevents dragging controls completely off-screen
- User can still position anywhere within viewport
- Avoids "lost controls" scenario

---

### **Issue 3: Keyboard Event Bubbling**

**Problem:**
- Shift + D listener on window may capture keystrokes in input fields

**Current Fix:**
```tsx
if (e.shiftKey && e.key === 'D') {
  e.preventDefault(); // Prevents default behavior
  // ...
}
```

**Potential Issue:**
- `e.preventDefault()` may interfere with form inputs

**Better Fix:**
```tsx
if (e.shiftKey && e.key === 'D' && e.target === document.body) {
  e.preventDefault(); // Only prevent if not in input field
  // ...
}
```

**Rationale:**
- Only toggle controls when focus is on document body
- Allows typing "D" in input fields without toggling controls
