#### 🔴 Database-Level TTL Enforcement (CRITICAL)

**Requirement:** Ephemeral assets must be physically purged after 24 hours, leaving **ZERO TRACES**.

**Database Schema Analysis:**
```sql
CREATE TABLE ephemeral_assets (
  -- ... fields ...
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  expires_at TIMESTAMPTZ NOT NULL, -- created_at + INTERVAL '24 hours'
  deleted_at TIMESTAMPTZ,
  -- ...
);

-- Row-Level Security (CRITICAL)
ALTER TABLE ephemeral_assets ENABLE ROW LEVEL SECURITY;

CREATE POLICY ephemeral_assets_ttl_policy ON ephemeral_assets
  USING (expires_at > NOW() AND deleted_at IS NULL);
```

**✅ COMPLIANT:**
- TTL trigger auto-sets `expires_at` to `created_at + 24 hours` ✅
- RLS policy **BLOCKS READS** of expired assets at database level ✅
- Expired assets become invisible to ALL queries ✅

**⚠️ PENDING:**
- [ ] **Physical deletion:** Cron job marks as `deleted_at` but does NOT physically delete rows
- [ ] **Storage deletion:** `asset_url` files must be deleted from S3/Supabase Storage
- [ ] **Metadata traces:** Row still exists in table after 24h (marked as deleted)

**Recommendation:**
1. Modify `purge_expired_ephemeral_assets()` to PHYSICALLY delete rows:
   ```sql
   DELETE FROM ephemeral_assets
   WHERE expires_at < NOW() - INTERVAL '1 hour'; -- Grace period
   ```

2. Add storage deletion:
   ```typescript

   ```

